# prompt:manage

입력된 프롬프트를 Best Practice 6축으로 분석 → 최소 3가지 질문 → 개선된 프롬프트 출력.

---

## Best Practice 분석 기준 (내부 참조)

| 축 | 점검 항목 | 누락 시 질문 방향 |
|----|-----------|------------------|
| 1. 역할/컨텍스트 | 도메인, 페르소나, 배경 명시 여부 | "어떤 역할/전문성을 가진 사람이 답해주길 원하나요?" |
| 2. 출력 형식 | 형식(표/코드/글), 길이, 언어 명시 여부 | "결과물의 형식과 분량은 어떻게 원하나요?" |
| 3. 제약 조건 | 금지 사항, 범위, 기술 스택 제한 | "변경하면 안 되는 것이나 사용해야 할 기술이 있나요?" |
| 4. 예시 | 입력/출력 예시 유무 | "기대하는 결과물의 예시를 하나 들어줄 수 있나요?" |
| 5. 추론 방식 | 단계적 사고 요청 여부 (CoT) | "중간 추론 과정을 보여줄 필요가 있나요?" |
| 6. 목표 명확성 | 최종 목적 vs 수단 혼동 여부 | "이 결과물로 궁극적으로 무엇을 하려고 하나요?" |

---

## 워크플로우

### Step 1 — 프롬프트 입력 받기

사용자가 개선할 프롬프트를 제공하지 않은 경우 요청:

> "개선하고 싶은 프롬프트를 붙여넣어 주세요."

### Step 2 — 분석 및 질문 생성

원본 프롬프트를 위 6축 기준으로 분석한다.
누락/모호한 축을 파악하고, **임팩트가 높은 순서로 최소 3개, 최대 5개** 질문을 선택한다.

출력 형식:

```
## 프롬프트 분석 결과

원본을 분석했습니다. 더 좋은 결과를 위해 아래를 확인할게요.

**Q1. [질문 제목]**
→ [구체적 질문 내용]
(답변 예시: "...")

**Q2. [질문 제목]**
→ [구체적 질문 내용]
(답변 예시: "...")

**Q3. [질문 제목]**
→ [구체적 질문 내용]
(답변 예시: "...")

단답 OK — "1: 답변, 2: 답변, 3: 답변" 형식으로 답해도 됩니다.
```

### Step 3 — 개선된 프롬프트 출력

사용자 답변을 반영하여 개선된 프롬프트를 생성한다.

출력 형식:

```
## 개선된 프롬프트

---
[개선된 프롬프트 전문]
---

## 주요 변경 사항
- **[축 이름]**: [원본에서 무엇이 추가/수정되었는지]
```

---

## 주의사항

- 질문은 임팩트 순으로 선택한다 — 모든 축을 무조건 질문하지 않는다
- 원본 프롬프트의 의도와 핵심은 보존한다 — 내용을 바꾸지 않는다
- 개선된 프롬프트는 그대로 복사해서 쓸 수 있게 완성도 있게 출력한다
- 사용자가 질문을 건너뛰고 싶으면 "스킵" 또는 빈 답변 허용 → 해당 축은 원본 유지
