# learn:growth-maker

SOCRAAI DevOps 엔지니어 성장 플래닝 워크플로우. 3가지 모드로 동작한다.

---

## 핵심 원칙

- **SOCRAAI 연결 필수**: 모든 추천은 SOCRAAI 인프라(아래 도메인 지식 맵)와 연결. 범용 추천 금지
- **팀 기조 반영**: 생산성 > 비용 > 안정성 가중치, Engineering over Money 지향
- **실행 위임**: 실제 딥다이브는 `/learn {topic}`으로 위임. 이 스킬은 메타 플래닝만 담당
- **글쓰기 연결**: 모든 학습 주제에 글쓰기 연결점 제시. 글쓰기 = 학습 완성

---

## 모드 선택

사용자 메시지에서 의도를 파악하여 모드를 자동 선택한다.

| 모드 | 트리거 의도 | 출력 |
|------|------------|------|
| **Analyze** | "뭘 공부해야 해?", "성장 방향", "갭 파악" | 현황 분석 + 우선순위 추천 (3-5개) |
| **Path** | "K8s 러닝 패스", "학습 순서", "단계적으로" | 의존성 그래프 + Phase별 계획 + `/learn` 링크 |
| **Write** | "글 뭐 쓰지?", "블로그 주제", "아티클 아이디어" | 주제 후보 → 선택 → 5단 아웃라인 |

모드가 불명확하면: "Analyze / Path / Write 중 어떤 것이 필요하신가요?" 질문 후 진행.

## Mode 1: Analyze

### 진행 방식

1. 현재 관심 도메인 또는 최근 작업한 영역을 사용자에게 질문 (1-2가지)
2. 도메인 지식 맵에서 연관 주제를 찾아 우선순위 결정
3. 아래 형식으로 출력

### 출력 형식

```
## 성장 분석

### 현재 위치
{사용자가 말한 맥락 기반 간단 요약}

### 추천 학습 주제

| 순위 | 주제 | SOCRAAI 연결 | 팀 기조 | `/learn` 명령 |
|------|------|------------|---------|--------------|
| 1 | {주제} | {구체적 연결} | 생산성/비용/안정성 | `/learn {topic}` |
| 2 | ... | ... | ... | ... |

### 추천 이유
{상위 1-2개에 대해 왜 지금 이 주제인지 2-3문장으로 설명}

### 글쓰기 연결
{학습 후 어떤 글로 발전시킬 수 있는지 1줄씩}
```

## Mode 2: Path

### 진행 방식

1. 목표 도메인 확인 (K8s / AWS / CS / Observability 중)
2. 현재 수준 확인 (beginner / intermediate / advanced)
3. 도메인 지식 맵에서 의존성 관계 도출 → Phase 설계

### 출력 형식

```
## 러닝 패스: {도메인}

### 의존성 그래프
{주제 A} → {주제 B} → {주제 C}
              ↓
           {주제 D}

### Phase별 계획

#### Phase 1 — {기초 주제}
| # | 주제 | 난이도 | SOCRAAI 연결 | `/learn` 명령 |
|---|------|--------|------------|--------------|
| 1 | ... | beginner | ... | `/learn ...` |

#### Phase 2 — {심화 주제}
| # | 주제 | 난이도 | SOCRAAI 연결 | `/learn` 명령 |
|---|------|--------|------------|--------------|
| 1 | ... | intermediate | ... | `/learn ...` |

#### Phase 3 — {실전 적용}
{우리 환경에서 실제로 적용/개선할 수 있는 것}

### 글쓰기 마일스톤
- Phase 1 완료 후: {글 주제 제안}
- Phase 2 완료 후: {글 주제 제안}
```

## Mode 3: Write

### 진행 방식

1. 최근 학습하거나 실무에서 다룬 주제를 질문
2. 도메인 지식 맵 기반 글쓰기 주제 후보 5개 생성
3. 사용자가 번호 선택 → 5단 아웃라인 출력

### 주제 후보 형식

```
## 글쓰기 주제 후보

| # | 제목 | 핵심 가치 | 난이도 |
|---|------|----------|--------|
| 1 | {구체적 제목} | {독자에게 줄 인사이트} | 입문/중급/심화 |
| 2 | ... | ... | ... |

번호를 선택하면 상세 아웃라인을 작성합니다.
```

### 아웃라인 형식 (선택 후)

```
## 아웃라인: {제목}

### 1. 도입 (Hook)
{독자를 끌어들이는 문제 상황 또는 반전 포인트}

### 2. 배경 (Problem)
{기존 방식의 한계, 왜 이 문제가 중요한가}

### 3. 접근 (Approach)
{우리가 선택한 방법, 핵심 기술 설명}

### 4. 결과 (Results)
{수치, 개선 전후 비교, 트레이드오프}

### 5. 교훈 (Lessons)
{팀 기조와 연결, 다음에 더 잘할 것}

### SOCRAAI 연결
{우리 환경의 어떤 구체적 경험에서 나왔는지}
```

- [ ] 모든 추천에 SOCRAAI 연결점 존재 (도메인 지식 맵에서 찾지 못하면 명시)
- [ ] Path 모드: `/learn {topic}` 링크가 각 주제마다 포함됨
- [ ] Write 모드: 아웃라인에 "SOCRAAI 연결" 섹션 포함됨
- [ ] 추천 근거가 팀 기조(생산성/비용/안정성/엔지니어링)에 연결됨
