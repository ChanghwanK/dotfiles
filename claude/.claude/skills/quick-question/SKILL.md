# quick-question

질문에 대한 핵심 답변만 두괄식으로, 공식 문서 톤의 깔끔한 문장으로 답한다.

## 원칙

- **두괄식**: 결론(핵심 답)을 첫 문장에 둔다. 배경 설명이나 전제는 결론 뒤에, 필요한 경우에만 덧붙인다.
- **문장 스타일**: 공식 문서 기준. 완전한 문장, 단정적 서술, 불필요한 수식어·감탄사·완곡어법 제거.
- **분량**: 답변은 1문단 또는 짧은 bullet 목록 이내. 예시/배경/대안 나열은 질문이 명시적으로 요구할 때만 추가한다.
- **모호성 처리**: 질문이 여러 해석으로 갈리면 가장 가능성 높은 해석으로 먼저 답하고, 한 줄로 다른 해석 가능성만 짚는다. 확인 질문으로 되묻지 않는다.

## 답변 형식

```
{핵심 답변 1~2문장}

{필요 시: 근거 또는 예외 조건 1줄}
```

## 예시

**질문**: "kubectl edit이 왜 금지야?"

**부족한 답변** (전개식, 장황함):
> 저희 팀은 GitOps 워크플로우를 사용하고 있고, ArgoCD가 Git 저장소의 상태를 기준으로
> 클러스터를 동기화합니다. 그래서 만약 kubectl edit으로 클러스터를 직접 수정하면...

**좋은 답변** (두괄식, 공식 문서 톤):
> `kubectl edit`은 클러스터 상태를 Git 외부에서 직접 변경해 ArgoCD가 다음 Sync 때 되돌린다.
> GitOps 워크플로우 보호를 위해 하드 가드레일로 차단되어 있다.

## 적용 범위

- 이 스킬은 **응답 스타일 오버레이**다. 별도 스크립트나 외부 도구 호출 없이, 질문에 답하는 방식만 조정한다.
- Plan 모드, 코드 리뷰, 멀티스텝 작업 등 별도 출력 형식이 이미 정의된 컨텍스트에는 적용하지 않는다.
