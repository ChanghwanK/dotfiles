# 이력서 Work Experience 포맷 표준

이력서 work experience 항목(bullet)의 표준 작성 포맷이다. `task:review` 스킬의 "이력서 bullet" 출력과
`notion-review` 에이전트가 Task 문서를 리뷰할 때 이 기준을 함께 참조한다.

경력기술서/성과평가용 문장(PAR 구조: Problem/Action/Result 레이블 + 서술형)은 이 표준의 적용 대상이
**아니다**. 그쪽은 기존 PAR 포맷을 그대로 유지한다.

## 핵심 구조

한 줄에 **문제(정량 before) → 행동(기술/방법) → 결과(정량 after)** 를 압축하고, **명사형으로 종결**한다.

```
{Situation/원인}(으)로 {Before 수치가 있으면 포함하여} {현상}을/를,
{핵심 Action: 구체적 방법·기술}(으)로 {Result: 화살표 또는 대괄호 표기} {명사형 종결}
```

## 규칙

- **명사형 종결 필수**: "~했습니다", "~합니다" 같은 문장형 종결을 쓰지 않는다. 개선/해결/절감/구현/전환/완료
  같은 명사형으로 끝맺는다.
- **Before 수치를 Problem 절에 포함**: 가능하면 "[N건/월] 발생하던"처럼 문제 발생 시점의 수치를 함께 적는다.
  Before 수치가 없으면 생략하되, After 수치(Result)는 반드시 있어야 한다. 정성 결과만 있을 때도 검증
  가능한 표현("추적 가능성 확보", "롤백 경로 확보")을 쓰고 막연한 표현("잘 됨", "개선됨")은 쓰지 않는다.
- **Result는 대괄호 또는 화살표로 표기**: `[0건 / N% 감소]`, `[N분→M분]`처럼 시각적으로 분리해 가독성을
  높인다.
- **기술명은 Action의 근거로만 사용**: 기술 나열이 아니라 "왜 그 방법을 택했는지"가 드러나야 한다.
- **1줄 압축, 최대 2줄**: 긴 배경 설명은 넣지 않는다.
- **실측 수치가 없을 때**: `[N건]`, `[N%]` 같은 플레이스홀더로 남기고, 실제 작성 시 반드시 실측치로
  치환해야 함을 표기한다. 가짜 수치를 채워 넣지 않는다.

## 예시

- "Nginx Ingress 재배치 시 preStop/SIGTERM 이슈로 [N건/월] 발생하던 5xx 에러를, preStop 드레이닝 방식
  및 NLB draining 설정 변경으로 [0건 / N% 감소]"
- "CronJob 지연으로 인한 데이터 유실을 gp2 burst 한계 기반 디스크 병목으로 진단, gp3 전환으로 유실
  [N건→0건] / 지연 [N분→M분] 해소"
- "AWS 비용 최적화를 위해 idle 리소스 rightsizing 및 Spot/RI 전환을 수행, 비용 [50% 절감
  ($100K/month)]"

## 나쁜 예 (수정 전)

- "Kubernetes 클러스터를 운영하고 OpenTelemetry Operator를 업그레이드함" (문제·결과 없이 활동만 나열)
- "~을 통해 보안을 개선했습니다" (문장형 종결, 결과 미정량)

## 적용 범위

- `task:review` 스킬 Part A.5 "이력서 bullet" 출력
- `notion-review` 에이전트가 task:review PAR 섹션(`### PAR 성과 문장`)이 포함된 Task 문서를 리뷰할 때
