# Notion 문서 작성 스타일

원칙: **짧고 단순하게, 핵심만.** 길고 복잡한 문장보다 짧고 읽기 쉬운 문장을 우선한다.

## 문장

- 격식체.
- 한 문장 한 메시지. 길면 끊는다.
- 군더더기를 빼고 핵심만. 쉬운 표현을 쓴다.
- 결론·핵심을 앞에 둔다.
- 능동태, 주어를 명확히.
- 산문은 한국어, 코드·명령어·식별자는 영어 원문을 유지한다.
- 한 개념엔 한 용어. 약어는 첫 등장 시 한 번 풀어쓴다.
- 모호어("잘 된다") 대신 측정 가능한 표현을 쓴다.

## 구조

- 긴 문단 대신 불릿으로 쪼갠다.
- 제목은 H1 → H2 → H3 순서로, 단계를 건너뛰지 않는다.
- 비교는 표, 열거는 불릿, 절차는 번호 목록.
- 볼드·콜아웃은 핵심에만. 인라인 코드는 식별자·값에.
- 긴 부차 정보(로그, 출력)는 토글로 접는다.
- 리스트 중첩은 2~3단계까지.
- 링크는 의미 있는 텍스트에 건다.

## 하드룰

- em dash(`—`) 금지. 콜론·쉼표·괄호로 대체한다.
- 본문에 이모지 금지.
- 둘 다 `notion-review` 에이전트가 작성 직후 자동 점검·수정한다.
